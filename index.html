页面数据如何对应真实 CI

main push 后的 Pages workflow 会读取合并提交中的 benchmark-metrics.json，生成 data/accepted-runs.json 并部署页面。PR 门控失败的分支不会合并，因此不会进入这个 JSON。

阶段一：分支内比较

B1 与 M1 比较。M1 是 PR 分支分叉点；如果 B1 在吞吐、TTFT 或失败数上不达标，说明 PR 自身引入了性能退化。

阶段二：最新 main 比较

CI 将 B1 本地 rebase 到 M2 得到 B1′，再比较 B1′ 与 M2。这样可以发现 main 更新后 PR 是否仍然满足性能门槛。

人工处理边界

CI 只在本地 rebase 用于测量，不会把 B1′ 推回远端。rebase 冲突或性能退化需要开发者修复后重新触发 PR 检查。
