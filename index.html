How this maps to the real CI

Stage 1: branch-local gain

B1 is compared with M1, the base commit where the branch diverged. This proves the PR branch improved over its own starting point.

Stage 2: current-main gain

CI locally rebases B1 onto current main M2 to create B1′, then compares B1′ with M2. This proves the optimization still helps on latest main.

Manual intervention boundary

CI may rebase locally for measurement, but it should not push B1′. Rebase conflicts or performance regressions are fixed manually by the developer.
