How this maps to the real CI

The real PR workflow can use the same structure for the GitHub summary/comment while the main branch keeps the formal 14B same-spec leaderboard flow.

Stage 1: branch-local gain

B1 is compared with M1, the commit where the branch diverged. This proves the PR branch improved over its own starting point.

Stage 2: current-main gain

CI locally rebases B1 onto current main M2 to create B1′, then compares B1′ with M2.

Manual intervention boundary

CI can rebase locally for measurement, but should not push B1′. Conflicts or regressions are fixed by the developer.
